## Decision tree

### Step 1 — Drain `seochecksiteToDo.txt`
For each non-header line below the `---`:
- `URGENT:` prefix → add to `URGENT_FOR_COWORK.md` Open + act inline if quick
- `QUESTION:` prefix → answer in chat to Michael; delete the line after
- Otherwise → act directly OR add to `TEAM_BOARD.md` Backlog with a fresh D-ID, then delete
- ALWAYS re-read the file before deleting; he may add concurrently

### Step 2 — Process `URGENT_FOR_COWORK.md` Open items
- Resolve directly if possible; move to Resolved
- If it needs Hermes → write directive in `checksite_cowork_to_hermes.md`
- If it needs Codex → write review request in `checksite_cowork_to_codex.md`
- If gated (stop-and-ask list below) → surface to Michael with yes/no + default

### Step 3 — Process latest Hermes reply
- ✅ DONE → request Codex review (R-XXX), update TEAM_BOARD row to 🔵 REVIEW
- 🟡 PARTIAL → write D-XXXa follow-up clarifying or unblocking
- 🔴 BLOCKED → check gates; if not gated, propose path; if gated, surface
- 🔄 IN PROGRESS → no action; let Hermes work

### Step 4 — Process latest Codex verdict
- ✅ PASS → mark TEAM_BOARD row 🟢 DONE; log one-liner to REVIEWS_FROM_CODEX.md
- 🟡 PARTIAL → write Hermes directive to fix named gap; log verdict
- 🔴 FAIL → write Hermes directive with failure evidence; log; surface if gate-touching

### Step 5 — If everything is quiet, do at most ONE proactive thing
Only if there's a real signal, otherwise return "no findings":
- Read most recent customer audit report (via Supabase `audits` table) — look for new failure patterns
- Spot-check one module score on a recent audit against the customer's actual site
- Review last deploy's error logs
- Pull `funnel_events` last 24h — look for conversion drops
- Audit `TEAM_BOARD.md` Backlog — promote one stale-but-valuable item

Write the finding to `URGENT_FOR_COWORK.md` (if time-sensitive) or `TEAM_BOARD.md` Backlog (if not). Never more than one finding per poll.

### Step 6 — Last resort: ask Michael in chat
With a specific yes/no question + recommended default.
